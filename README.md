# Simon Express — simonexpress.com

National refrigerated trucking carrier website. Static HTML deployed on Vercel,
form submissions handled by a Vercel edge function via Resend.

## Stack
- Static HTML / CSS / JS (no build step at deploy time)
- Vercel hosting
- One serverless edge function: `api/submit-form.js` (Resend email)
- All pages pre-rendered locally via `build_pages.py`

## Pages
- `index.html` — Home
- `services.html` — Refrigerated services overview
- `fleet.html` — Equipment / fleet detail
- `about.html` — Company story
- `careers.html` — CDL hiring (with JobPosting + FAQ schema)
- `driver-app.html` — Driver app marketing
- `contact.html` — Contact + dispatch info
- `quote.html` — Freight quote request form
- `tracking.html` — Customer tracking (Coming Soon placeholder)
- `driver-login.html` — Driver portal login (noindex; stub form, wire to auth later)

## Deploy to Vercel

### First-time setup

1. Create a new GitHub repository, push this folder to it
2. In Vercel, click **Add New → Project**, import the GitHub repo
3. Framework preset: **Other**
4. Build command: leave blank
5. Output directory: leave blank (root)
6. Add environment variables (see below)
7. Click **Deploy**

### Required environment variables

Set these in Vercel project settings → Environment Variables:

```
RESEND_API_KEY    = re_xxxxxxxxxxxxxxxxxxxx    # from resend.com
FROM_EMAIL        = forms@simonexpress.com     # must be a verified Resend sender
TO_EMAIL          = info@simonexpress.com      # where form submissions land
```

Without these vars set, forms will fall back to a `mailto:` flow.

### Connect domain
1. In Vercel project → Settings → Domains, add `simonexpress.com` and `www.simonexpress.com`
2. Update DNS records as instructed by Vercel
3. Set `www.simonexpress.com` as the primary

## Local development

```bash
# Open any HTML file directly in a browser, or:
python3 -m http.server 8000
# then visit http://localhost:8000
```

To regenerate subpages from `build_pages.py`:

```bash
python3 build_pages.py
```

This pulls shared partials (utility bar, nav, footer, base schema) from `index.html`
and rewrites all subpages.

When editing the home page, edit `index.html` directly. When editing any subpage,
edit the corresponding section in `build_pages.py` and run the build.

## Driver login page

`driver-login.html` is a placeholder form. The submit button currently shows an
alert and does NOT log anyone in. To wire it up to real auth:

1. Replace the inline `onsubmit` handler in `build_pages.py` (look for
   `DRIVER_LOGIN_BODY`) with your auth POST endpoint
2. On successful auth, redirect drivers to your driver-app URL
3. The page is `noindex, nofollow` and excluded from `robots.txt` so search
   engines won't crawl it

## SEO / AEO notes

The site is heavily optimized for AI answer engines (ChatGPT, Perplexity, Claude,
Google AI Overviews). Key features:

- Atomic answer paragraphs at the top of every page (40-60 words, AI-extractable)
- Organization + LocalBusiness schema with award array (Best Fleets nominations)
- FAQPage schema on home, services, careers, contact (matches visible HTML)
- Comprehensive JobPosting schema on careers page
- WebSite + per-page-specific schema (Service, AboutPage, ContactPage)
- Sitemap, robots.txt, canonical URLs, OG/Twitter meta on every page

When a driver or shipper asks an AI tool "what's a good refrigerated trucking
company?" or "what's it like to drive for Simon Express?", these schemas + atomic
answers are what the AI extracts and cites.
